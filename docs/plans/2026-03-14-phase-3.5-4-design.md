# Phase 3.5 + Phase 4 Design — ICT Indicators, Strategies & AI Infrastructure

**Date:** 2026-03-14
**Status:** Approved

## Context

Phase 3B (Backtest Realism) is complete. Two bodies of work remain:
- **Phase 3.5** — 54 ICT/SMT indicators + 19 ICT strategies across 7 indicator modules + 19 strategy files
- **Phase 4** — AI Research Agents (regime detection, strategy lifecycle, Optuna robustness, API endpoints, n8n workflow updates)

Execution is interleaved: indicators -> AI infrastructure -> strategies -> n8n.

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Strategy architecture | Unified `BaseStrategy` protocol | One interface for expression + ICT strategies. Backtester has one code path. |
| Expression strategies | Wrapped in `ExpressionStrategy(BaseStrategy)` | Backward compatible. 5-indicator cap stays here only. |
| `status` column | Replace with `lifecycle_state` | One source of truth. Map: draft=CANDIDATE, active=DEPLOYED, archived=RETIRED. |
| Ollama model | `qwen3-coder:30b` | Available on machine. |
| Optuna | Install if missing | Required for robustness testing. |
| Strategy file layout | 19 separate files (one per strategy) | Clean diffs, parallel agent work, instant lookup by filename. |
| ICT indicator cap | Bypassed for class-based strategies | ICT needs 5+ indicator types by nature but keeps max 3-5 tunable params. |
| Job tracking | Reuse `audit_log` table | Already tracks action, status, input, result, duration. No new table needed. |

## Section 1: Unified Strategy Interface

**New file: `src/engine/strategy_base.py`**

```python
from abc import ABC, abstractmethod
import polars as pl

class BaseStrategy(ABC):
    """All strategies implement this. Backtester only knows this interface."""

    name: str
    symbol: str
    timeframe: str
    preferred_regime: str | None = None

    @abstractmethod
    def compute(self, df: pl.DataFrame) -> pl.DataFrame:
        """Add indicator columns + signal_long/signal_short/signal_exit bool columns."""
        ...

    @abstractmethod
    def get_params(self) -> dict:
        """Return tunable params for optimizer. Max 5 for expression strategies."""
        ...

    @abstractmethod
    def get_default_config(self) -> dict:
        """Return default param values for serialization/DB storage."""
        ...
```

**`ExpressionStrategy`** wraps existing `StrategyConfig`:

```python
class ExpressionStrategy(BaseStrategy):
    def __init__(self, config: StrategyConfig):
        self.config = config
        self.name = config.name
        self.symbol = config.symbol
        self.timeframe = config.timeframe

    def compute(self, df: pl.DataFrame) -> pl.DataFrame:
        result = compute_indicators(df, self.config.indicators)
        result = result.with_columns([
            evaluate_expression(result, self.config.entry_long).alias("signal_long"),
            evaluate_expression(result, self.config.entry_short).alias("signal_short"),
            evaluate_expression(result, self.config.exit).alias("signal_exit"),
        ])
        return result
```

**Backtester change:** Replace indicator+expression block with `strategy.compute(df)`. Accept `BaseStrategy` or `StrategyConfig` (auto-wrapped).

Indicator functions are composable:
```
detect_swings(df) -> swings
    ├── detect_bos(df, swings)
    ├── detect_choch(df, swings)
    ├── detect_bullish_ob(df, swings)
    ├── detect_buyside_liquidity(df, swings)
    └── auto_swing_fib(df, swings)

detect_fvg(df) -> fvgs
    ├── detect_ifvg(df, fvgs)
    ├── compute_consequent_encroachment(fvgs)
    └── detect_propulsion(df, obs, fvgs)
```

ICT strategies call indicator functions directly — NOT registered in `VALID_INDICATOR_TYPES` or dispatcher.
