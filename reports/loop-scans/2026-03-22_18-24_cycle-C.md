## Agent 1: Strategy Files (19 files scanned)

### Confirmed Bugs — FIXED

**BUG P2: unicorn.py:160-221 — Same-bar entry+exit: no exited_this_bar guard**
- Exit logic can fire `exit_long[i]=True` and reset `long_entry_bar=-1`. Entry logic then sees flat state and fires `entry_long[i]=True`. Both signals on same bar → vectorbt silently drops the trade.
- **Fix:** Added `exited_this_bar_long`/`exited_this_bar_short` flags. Entry checks now include `not exited_this_bar_*` guard, matching the pattern used in 13 other strategies.

**BUG P2: power_of_3.py:162-204 — Same-bar entry+exit: no exited_this_bar guard**
- Same structural issue as unicorn. Exit resets entry_bar, entry immediately re-fires.
- **Fix:** Added `exited_this_bar_long`/`exited_this_bar_short` flags with guards on entry conditions.

**BUG P2: eqhl_raid.py:147-174 — Missing mutual exclusion between long and short**
- Long entry checks `not in_long` but not `not in_short`. Short entry checks `not in_short` but not `not in_long`. If `choch == "bullish"` AND `mss == "bearish"` on same bar, both fire simultaneously.
- **Fix:** Added `and not in_short` to long entry, `and not in_long` to short entry.

**BUG P2: ict_swing.py:192-229 — Position flip orphans prior position in vectorbt**
- When `entry_short` fires while `in_long=True`, code sets `in_long=False` but never emits `exit_long[i]=True`. vectorbt's long channel has no exit signal → position persists until data end.
- **Fix:** Added `exit_long[i]=True` when flipping long→short, `exit_short[i]=True` when flipping short→long.

### Noted — Not Fixed (P3)

| Bug | File:Line | Issue | Why Deferred |
|-----|-----------|-------|-------------|
| P3 | quarterly_swing.py:248 | Long and short can be simultaneously active | No crash/data corruption, session exit closes both, low frequency |

- `compute_premium_discount` and `compute_equilibrium` use `<=` for swing pointer advancement. `detect_bos` and `detect_choch` use `<`.
