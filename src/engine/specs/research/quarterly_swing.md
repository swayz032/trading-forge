## Consensus Rules (all sources agree)

### Core Principle
Every time period divides into 4 equal segments with predictable behavioral phases. The phases map to ICT's Power of 3 plus a fourth phase:

| Quarter | Phase | Behavior | Tradeable? |
|---------|-------|----------|------------|
| Q1 | Accumulation | Consolidation, range-bound, smart money building positions | Low probability — establishing range |
| Q2 | Manipulation | Judas Swing / false breakout, sweeping liquidity | Setup phase — identifies direction |
| Q3 | Distribution | Real move, highest volatility, smart money distributing | **PRIMARY TRADE PHASE** |
| Q4 | Continuation/Reversal | Follow-through or reversal into next cycle | Secondary / management phase |

### Time Divisions (all in ET/New York)

#### Daily Quarters (6-hour sessions)
| Quarter | Session | Time (ET) | Phase |
|---------|---------|-----------|-------|
| Q1 | Asia | 18:00 - 00:00 | Accumulation |
| Q2 | London | 00:00 - 06:00 | Manipulation |
| Q3 | NY AM | 06:00 - 12:00 | Distribution |
| Q4 | NY PM | 12:00 - 18:00 | Continuation |

#### Weekly Quarters
| Quarter | Day | Phase |
|---------|-----|-------|
| Q1 | Monday | Accumulation |
| Q2 | Tuesday | Manipulation |
| Q3 | Wednesday | Distribution (highest range) |
| Q4 | Thursday | Continuation |

Note: Friday is excluded from the weekly cycle — it has its own dynamics (position squaring, weekend risk).

#### 90-Minute Session Quarters
Each 6-hour session subdivides into four 90-minute quarters with the same AMD pattern.

### True Open
- The True Open is the opening price at the START of Q2 in any cycle
- Daily True Open = London session open (00:00 ET) or more precisely 01:30 ET per Daye
- Weekly True Open = Tuesday open
- **Trading rule:** Bullish setups enter below the True Open; bearish setups enter above it

### Entry Model (consensus from all sources)
1. **Identify bias:** Use higher timeframe (HTF) structure to determine bullish or bearish bias
2. **Wait for Q2 manipulation:** Price makes a false move (Judas swing) — sweeps liquidity in the WRONG direction
3. **Enter during Q3:** After manipulation completes, enter in the REAL direction during the distribution phase
4. **Confirm with:** Displacement, FVG, MSS/BOS, or order block entry within Q3
5. **True Open reference:** For longs, price should be below True Open before entry; for shorts, above it

### Exit Model
- **Primary exit:** Target opposite liquidity level or prior session high/low
- **Secondary exit:** Phase transition into Q4
- **Stop loss:** Beyond the manipulation extreme (Q2 swing high/low)

| **Q3 vs Q2-Q3 entry window** | Most say Q3 only; some allow late Q2 entries | Recommend: primary entries in Q3, allow late Q2 if manipulation is clearly ending |

## Critical Validation Findings vs Current Strategy Code

### ISSUE 1: quarterly_theory() function is WRONG
**Current:** Divides each HOUR into four 15-minute quarters.
**Correct:** Divides the TRADING DAY into four 6-hour sessions (Asia/London/NYAM/NYPM).
**Severity:** CRITICAL — the entire phase detection is operating at the wrong timeframe.

### ISSUE 2: Entry phases are inverted
**Current:** Enters longs during Q1 (accumulation) and Q2 (manipulation) in discount.
**Correct:** Accumulation and manipulation are SETUP phases, not entry phases. Entries should happen during Q3 (distribution) — the real move.
**Severity:** CRITICAL — strategy enters during the wrong market phases.

### ISSUE 3: Missing True Open reference
**Current:** Uses premium/discount zones from HTF swings.
**Correct:** ICT Quarterly Theory uses the True Open (Q2 opening price) as the key reference level. Longs enter below True Open, shorts enter above.
**Severity:** HIGH — premium/discount is a valid ICT concept but not the primary framework for Quarterly Theory.

### ISSUE 4: Missing liquidity sweep / manipulation confirmation
**Current:** Only requires BOS + premium/discount.
**Correct:** The Q2 manipulation (Judas swing) should sweep liquidity before Q3 entry. This is the core setup — without it, there's no confirmation that manipulation occurred.
**Severity:** HIGH — missing a key confluence filter.

### ISSUE 5: Exit on phase transition is partially correct
**Current:** Exits on any phase transition.
**Correct:** Exiting when transitioning from Q3 to Q4 is correct. But the strategy should NOT exit on Q1->Q2 or Q2->Q3 transitions if it has a position from the correct phase.
**Severity:** MEDIUM — exit logic needs refinement.

### VERDICT: FAIL — strategy requires rewrite
